# Community

## Where to ask for help

- Usage questions and bug reports: GitHub Issues
- Security concerns: follow the private reporting instructions in `SECURITY.md`

## How to contribute effectively

- Read `CONTRIBUTING.md` before opening a pull request.
- Include reproducible steps for bugs and test evidence for code changes.
- Keep pull requests focused and scoped to one concern when possible.

## Maintainer expectations

- Maintainer response targets and governance details are documented in
  `MAINTAINERS.md` and `GOVERNANCE.md`.
