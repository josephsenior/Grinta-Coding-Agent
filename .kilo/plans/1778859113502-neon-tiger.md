# Plan: Fix text_editor range-mode missing-parameter error

## Problem

When the text_editor tool is invoked with `edit_mode=range` but without `end_line` in the JSON call, the error returned is:

```
ERROR: edit_mode=range requires start_line and end_line.
```

This message is misleading because:
1. It says the tool "requires start_line and end_line" even when only `end_line` is missing (the user DID provide `start_line`)
2. It does not name which parameter is actually absent
3. There is **no early validation** at action-creation time (in `_handle_text_editor_tool`) for the `edit_mode=range` required-parameter combination, unlike `edit_mode=format` and `edit_mode=section` which each have dedicated guard checks that return specific, actionable errors

## Root Cause

Two places need fixing in `backend/engine/function_calling.py`:

### 1. `_handle_text_editor_tool` (line 453)
After `edit_mode` is validated, there is no guard that checks `start_line` / `end_line` are present when `edit_mode='range'`. The user gets no feedback until the action runs deeply in the execution stack, by which point a generic `None or None` check fires.

### 2. `branch_range` in `file_editor_edit_ops.py` (line 116-122)
The guard condition `if start_line is None or end_line is None:` is correct logically, but the error string says `"edit_mode=range requires start_line and end_line."` â€” it says **both** are missing without naming which one â€” even when only `end_line` is absent.

## Changes

### File: `backend/engine/function_calling.py`
After line 453 (after the `edit_mode not in valid_edit_modes` guard), add early validation for the range edit required parameters â€” exactly mirroring the pattern already used by `branch_format` / `branch_section` in `file_editor_edit_ops.py`:

```python
        # Early validation for edit_mode=range required parameters
        if edit_mode == 'range':
            if normalized_args.get('start_line') is None:
                raise FunctionCallValidationError(
                    "edit_mode=range requires 'start_line'. "
                    "Example: {'command': 'edit', 'edit_mode': 'range', 'start_line': 1, 'end_line': 10, 'new_str': '...'}"
                )
            if normalized_args.get('end_line') is None:
                raise FunctionCallValidationError(
                    "edit_mode=range requires 'end_line'. "
                    "Example: {'command': 'edit', 'edit_mode': 'range', 'start_line': 1, 'end_line': 10, 'new_str': '...'}"
                )
```

### File: `backend/execution/utils/file_editor_edit_ops.py`
Improve the `branch_range` guard error message (lines 116-122) to explicitly name the missing parameter rather than stating both are required:

```python
    def branch_range() -> str | Any:
        missing = []
        if start_line is None:
            missing.append('start_line')
        if end_line is None:
            missing.append('end_line')
        if missing:
            plural = 's' if len(missing) > 1 else ''
            param_list = ' and '.join(f"'{p}'" for p in missing)
            return _tool_result(
                output='',
                error=f'edit_mode=range requires {param_list}.',
                new_content=old_content,
            )
```
